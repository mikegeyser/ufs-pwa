Performance

"The app loads in under a second..."
What does 'loaded' mean?

  1. Is it happening?
    - First paint (FP)
  2. Is it useful?
    - First meaningful paint (FMP)
  3. Is it usable?
    - Time to interactive (TTI)
JavaScript is your most expensive asset.
- Alex Russell

Script loading delays TTI

TTI Goals:

  • < 5s first load
  • < 2s subsequent


Budget: ± 170kb

The PRPL Pattern

  • Push critical resources.
    - HTTP/2, or link rel="preload"
  • Render initial route.
    - Minimal AppShell
  • Pre-cache remaining routes.
    - Precache using Service Worker
  • Lazy-load on demand.
    - Code splitting

Why should I care?

Mobile

± 5 Billion Devices Connected to the Web

(More mobile than desktop)

13% of time on mobile is on web, 87% on native apps.

Use about 10 apps per day

80% of time is spent in 3 apps

On average, a user installs ZERO new apps per month

Average app size is ±15mb Android and ± 38mb iOS

The future is mobile.

(but the audience is picky)

Web

90% of sites are sending ~1mb gzipped JS

(~ 4mb, 4s on parse and compile)

40% of JS downloaded is unused

Average 35s TTI on mobile

After 3s, 20% of users abandon the session

Wasteful and lazy!

(and it's hurting us badly)
